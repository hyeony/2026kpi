# CUPPIN

## 1. 한 줄 소개

**팀원별 커피 취향을 미리 저장해 두고, 오늘 주문할 사람만 고르면 음료·주문 문구가 자동으로 완성되는 사내 커피 주문 도구**

---

## 2. 기획 의도

### 배경 문제

- 회사에서 커피 주문 담당이 되면 **단톡·슬랙으로 일일이 “뭐 마실래?”** 를 돌려야 함
- 사람마다 음료·온도·디카페인 여부가 달라 **매번 확인·정리·오타**가 발생
- “지난번이랑 똑같이” 같은 반복 주문도 **기억에 의존**

### 해결 방향


| 기존            | CUPPCUPPIN             |
| ------------- | ---------------------- |
| 매번 취향을 물어봄    | **구성원 프로필에 취향 1회 저장**  |
| 주문 문구를 손으로 작성 | **선택만 하면 문구 자동 생성**    |
| 누가 뭘 마시는지 흩어짐 | **참여자별 + 음료별 집계** 한 번에 |


### 컨셉 

- **“누구랑 커피 마실래?”** 같은 소셜·동행 컨셉이 아님
- **“오늘 누구 커피 시킬까?”** — 주문 담당자의 **업무 효율**이 핵심

---

## 3. 타깃 사용자


| 역할               | 하는 일                                 |
| ---------------- | ------------------------------------ |
| **주문 담당자**       | 팀/회의 단위로 주문 목록 만들고, 문구 복사해 단톡·카페에 전달 |
| **팀원**           | 본인 음료 취향·메모를 프로필에 등록 (또는 담당자가 대신 관리) |
| **(간접)** 카페·바리스타 | 정리된 주문 문구로 빠르게 접수                    |


---

## 4. 핵심 가치

1. **취향 재사용** — 한 번 저장하면 매주 반복 입력 불필요
2. **선택만으로 주문 완성** — 음료·태그·메모가 자동 반영
3. **협업 마찰 감소** — 확인 횟수·정리 시간·실수 감소
4. **AI는 보조** — 사람 고르기·팀 묶기를 빠르게 (프로토타입은 규칙 기반 시뮬레이션)

---

## 5. 정보 구조 (프로토타입)

```
회사 (companyName)
 └── 구성원 (Profile)
      ├── 이름 · 부서
      ├── 선호 음료 (최대 2개)
      ├── 취향 태그
      └── AI 메모 (디카페인, 오후 주문 주의 등)

오늘의 주문 목록 (OrderSelection) ← 그룹이 아님, 당일 임시 목록
 ├── memberIds[]
 └── guests[] (1회성 게스트)
```

- **「그룹」은 저장되지 않음** — 오늘 누구 시킬지만 유지 (날짜 바뀌면 초기화)
- **즐겨찾기·최근 주문** — 데모용 샘플 데이터 (실제 저장·조합 저장 기능 없음)

| **프로필 드로어** | 내(또는 선택한 사람) 취향 수정 |

## 7. 사용자 플로우

### Flow A — 최초 셋업 (1회)

```
구성원 탭에서 팀원 확인
    ↓
헤더 프로필 클릭 → 취향 패널
    ↓
음료 선택 · 취향 태그 · AI 메모 입력
    ↓
프로필 저장 (localStorage)
```

### Flow B — 오늘 주문 만들기 (메인)

```
[방법 1] AI 입력
  "FE팀 주문 목록 만들어줘" → 멤버 목록 자동 구성

[방법 2] 바로 불러오기
  즐겨찾기 / CUPPIN원클릭

[방법 3] 직접 선택
  조직도 펼침 → 개별 또는 팀 전체 선택
  (+ 필요 시 게스트 1회 추가)
    ↓
선택된 인원의 저장 취향으로 음료 자동 채움
    ↓
하단 도크: N명 · M잔 · 음료 요약
    ↓
「주문 문구 복사」→ 단톡·카페에 붙여넣기
```

### Flow C — 주문 문구 형식 (자동 생성)

```
☕ {회사명} CUPPIN (날짜)

[참여자별]
• 홍길동: 아이스 아메리카노 (저카페인 선호)
...

[AI 취향 메모]
• 김철수: 디카페인만, 오후 주문 주의
...

[음료별 집계]
• 아이스 아메리카노 ×3 (홍길동, 이영희, ...)
...

총 N잔 · M명
```

## 8. 기능 상세

### 8.1 구성원 & 취향 관리

- 부서별 조직도 기반 선택 UI
- 음료 프리셋 + 직접 입력
- 취향 태그, AI 메모 (주문 문구에 반영)
- 메모 기반 태그 추천 (프로토타입)

### 8.2 주문 목록 구성

- AI 자연어 입력 (팀명·지난주 동일 등 — 규칙 매칭)
- 즐겨찾기 / 최근 주문 불러오기
- 조직도 직접 선택 (개인·팀 단위)
- 게스트 임시 추가
- 당일 선택 상태 localStorage 유지

### 8.3 주문 출력

- 참여자별 상세 + 음료별 집계
- 클립보드 복사 원클릭
- AI 인사이트 카드 (선택 구성 음료 패턴 요약)

### 8.4 프로토타입에서 아직 없는 것

- 사용자 조합 **즐겨찾기 저장**
- **실제** 최근 주문 이력 저장
- 서버·계정·권한
- 결제·카페 연동

## 9. 협업 효율 효과 (기대)


| Before              | After                   |
| ------------------- | ----------------------- |
| 단톡에 “뭐 마실래?” 10~30분 | 취향 이미 있음 → **선택만**      |
| 담당자가 엑셀·메모로 정리      | **자동 집계 문구**            |
| “나 저번이랑 같아” 반복 확인   | 프로필·즐겨찾기(향후)로 **재사용**   |
| 디카페인·메모 누락          | AI 메모 블록으로 **전달 누락 감소** |
| 회의마다 다른 담당자         | **동일 포맷**으로 인수인계 쉬움     |


**정량 예시 (가정)**

- 10명 팀 주문: 확인·정리 **15분 → 2~3분**
- 주문 실수(음료 착각): **구조화된 문구**로 감소

---

## 10. AI의 역할 (프로토타입)

- 실제 LLM이 아닌 **규칙 기반 시뮬레이션**
- 팀명·키워드로 멤버 필터링
- 응답 문구 + 인사이트로 **“똑똑한 비서” 느낌** 시연
- 향후: 자연어로 “디자인팀 빼고”, “게스트 1명 추가” 등 확장 가능

---

## 11. 설계 원칙

1. **주문 목록 ≠ 그룹** — 직접 선택은 오늘 주문 편집, 그룹 저장은 선택 기능
2. **취향은 사람에 귀속** — 조합이 아니라 프로필이 SSOT
3. **복사 한 번이 완료** — 기존 단톡·슬랙 워크플로우 유지
4. **모바일 퍼스트** — 담당자가 폰으로 빠르게 처리
